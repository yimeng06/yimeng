1. 值传递（没有&）：
void func(string str) { ... }
创建副本：调用时，实参会被复制一份传给函数
修改不影响原值：函数内对参数的修改不会影响调用者的原始数据
有性能开销：复制大对象（如长字符串）可能较慢

2. 引用传递（有&）：
void func(string& str) { ... }
直接访问原数据：传递的是原变量的引用（别名），不是副本
修改会影响原值：函数内修改参数会影响调用者的原始数据
无复制开销：性能更高，特别是对大对象

3. 常量引用传递（有const和&）：(只读)
void func(const string& str) { ... }  // 代码中使用的
最佳实践：结合了两种的优点
只读访问：不能修改原数据（因为有const）
无复制开销：性能高
安全：保护原始数据不被意外修改
